   5  ; SimpleID identity file.
   6  ;
   7  ; This file contains all the data associated with an identity.  It should
   8  ; always be named username.identity, where username is the user name to be used
   9  ; when logging into SimpleID.
  10  ;
  11  ; In this file, if a value contains non-numeric characters, you will need to
  12  ; surround it with quotation characters.
  13  ;
  14  
  15  ;
  16  ; The OpenID Identifier associated with this identity.  This is typically a
  17  ; URL, although the OpenID specifications allow the use of URIs and even XRIs.
  18  ;
  19  ; Relying parties must be able to resolve the identity to obtain the address
  20  ; of this SimpleID installation.
  21  ;
  22  ; WARNING: If you change the OpenID Identifier after you have used it in
  23  ; SimpleID, you will need to delete all files named 'identity-*.cache' in the
  24  ; cache directory.
  25  ;
  26  ; Examples:
  27  ;    http://example.com/
  28  ;    http://example.com:8888/
  29  ;    http://example.com/myopenid
  30  ;    https://example.com:8080/myopenid
  31  ;
  32  identity="http://example.com/"
  33  
  34  ;
  35  ; The password associated with this identity.  
  36  ;
  37  ; The password is encoded as follows:
  38  ;
  39  ;     pass="hash:algorithm:salt"
  40  ;
  41  ; There are three components to the password string.  Only the first component
  42  ; (the hash) is required, the other two are optional.
  43  ;
  44  ; 1. The hash of the password.  By default, the algorithm for hashing the
  45  ;    password is MD5.
  46  ;
  47  ;    There are various tools available which will give you the MD5 hash of any
  48  ;    given string.  If you have access to the command-line interface of PHP, you
  49  ;    can run the following command to give you the hash.
  50  ;
  51  ;    php -r "print md5('example password');"
  52  ;
  53  ;    Alternatively, you can upload the following PHP code to your web server, then
  54  ;    read off the hash with your web browser.  (This is recommended only if your
  55  ;    web server supports secure uploads (e.g. SFTP) and HTTPS, you that your
  56  ;    password is not sent over the Internet as clear text.)
  57  ;
  58  ;    <?php header('Content-Type: text/plain'); print md5('example password'); ?>
  59  ;
  60  ;    Alternatively, Wikipedia (http://en.wikipedia.org/wiki/MD5) has other resources
  61  ;    on generating this hash.
  62  ;
  63  ; 2. The algorithm used to hash the password.  If this is omitted, 'md5' is assumed.
  64  ;
  65  ;    Allowed algorithms are:
  66  ;
  67  ;    - md5
  68  ;    - sha1
  69  ;
  70  ; 3. The salt that is used to hash the password.  If the salt is not an empty
  71  ;    string, the password is appended by a colon character (:) then the salt
  72  ;    before a hash is calculated, that is:
  73  ;
  74  ;    hash(password:salt)
  75  ;
  76  ; Examples (these contain the same password):
  77  ;    ea07017619350413c8a0d604cffdbe50                   ; MD5 hash and no salt
  78  ;    81059d819510280a9a14c3ace78226159c70e3cf:sha1      ; SHA1 hash and no salt
  79  ;    5ed9216011934c1fecfe978c00c7aaf2::ideally-a-large-number-of-random-characters-to-use-as-salt                   ; MD5 hash with salt
  80  ;    0f491332ba8fc76f6157da05a62ff6e45a622990:sha1:ideally-a-large-number-of-random-characters-to-use-as-salt       ; SHA1 hash with salt
  81  ;
  82  pass="c8e24ae844b7cb9381g722ae8209a19a"

  85  ; Whether this user is given administrative privileges in SimpleID.
  86  ;
  87  ; This setting has no effect in the current version of SimpleID.  However,
  88  ; more functionality may be added to SimpleID in future versions which will
  89  ; be restricted to SimpleID administrators.

  99  ; Advanced users only: SSL client certificates associated with this identity.
 100  ;
 101  ; You can associate SSL client certificates to this identity, so that you can
 102  ; log in using certificates instead of supplying a user name or password.
 103  ;
 104  ; The SSL certicate is identified using two parameters:
 105  ;
 106  ; - the certificate's serial number
 107  ; - the distinguished name of the certificate's issuer
 108  ;
 109  ; You can find out these two values using OpenSSL by running the following
 110  ; commands (replacing the file name of the certificate as required):
 111  ; 
 112  ;    openssl x509 -noout -serial -in certificate.crt
 113  ;    openssl x509 -noout -issuer -in certificate.crt
 114  ;
 115  ; These two values are then joined together using a semicolon.
 116  ;
 117  ; This option is for advanced users only.  Please see the documentation at
 118  ; http://simpleid.koinic.net/documentation/advanced-topics/logging-using-client-ssl-certificates
 119  ; for details on how to set this up.
 120  ;
 121  ; Note, you must also enable the certauth extension in SimpleID.  To
 122  ; do this, make sure the SIMPLEID_EXTENSIONS option in config.php contains
 123  ; certauth
 124  
 125  ; Example:
 126  ;
 127  ;[certauth]
 128  ;cert[]="02A97C;/C=XX/O=Example CA/OU=Example CA Certificate Signing/CN=Example Client CA"
