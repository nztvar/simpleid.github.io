   3  Upgrading SimpleID
   4  ==================
   5  
   6  Introduction
   7  ------------
   8  
   9  SimpleID is currently in heavy development, with the software changing rapidly.
  10  As a result, it is very important to upgrade SimpleID when a new version is
  11  released.
  12  
  13  General information on upgrading can be found at
  14  <http://simpleid.koinic.net/documentation/getting-started/upgrading>.
  15  
  16  This document sets out the additional steps which may need to be performed to
  17  upgrade to a specific version of SimpleID.
  18  
  19  Upgrading to SimpleID 0.8
  20  -------------------------
  21  
  22  SimpleID 0.8 does not require any additional steps as part of the upgrade
  23  process.
  24  
  25  However, you will still need to run the upgrade script to complete the
  26  upgrade.  To run the script, use your web browser to go to
  27  
  28  http://www.exmaple.com/simpleid/upgrade.php
  29  
  30  where http://www.exmaple.com/simpleid/ is the URL of your SimpleID server
  31  (where you have moved the www directory).  You will need to be logged in as
  32  an administrator to proceed with the script.
  33  
  34  
  35  Upgrading to SimpleID 0.7
  36  -------------------------
  37  
  38  1. New storage framework
  39  
  40  SimpleID introduced a new storage framework.  The new framework allows SimpleID
  41  to store additional information about your identity.
  42  
  43  As a result, you will need to be aware that there is a new configuration option
  44  called SIMPLEID_STORE_DIR, which specifies the directory where this information
  45  will be stored.  This directory must exist and be readable and writable by the
  46  web server.
  47  
  48  If you are upgrading from SimpleID 0.6, this setting will default to the
  49  directory specified by SIMPLEID_CACHE_DIR in your config.inc.  However, it is
  50  a good idea to place this in a separate directory.  To do this, add the
  51  following line to your config.inc:
  52  
  53  define('SIMPLEID_STORE_DIR', '<your directory here>');
  54  
  55  2. Administrators
  56  
  57  SimpleID 0.7 introduced the concept of administrators.  Administrators
  58  have access to certain functions which regular users cannot.  To make a user
  59  an administrator, edit the user's identity file to include the following line:
  60  
  61  administrator=1
  62  
  63  3. User Interface Extension
  64  
  65  SimpleID 0.7 introduced support for the draft OpenID User Interface Extension.
  66  It is enabled by default for new installations of SimpleID, however if you
  67  are upgrading you may need to enable it manually.  To do so, edit
  68  your config.inc to change the SIMPLEID_EXTENSIONS configuration option to
  69  include ui.  For example:
  70  
  71  define('SIMPLEID_EXTENSIONS', 'sreg,ui');
  72  
  73  4. Upgrade script
  74  
  75  SimpleID introduced a new upgrade script.  This means that whenever you
  76  upgrade you will also need to run the script.  To run the script, use your
  77  web browser to go to
  78  
  79  http://www.exmaple.com/simpleid/upgrade.php
  80  
  81  where http://www.exmaple.com/simpleid/ is the URL of your SimpleID server
  82  (where you have moved the www directory).  You will need to be logged in as
  83  an administrator to proceed with the script.
  84  
  85  
  86  Upgrading to SimpleID 0.6
  87  -------------------------
  88  
  89  SimpleID version 0.6 introduced a new log in system.  The new system allows
  90  you to log in to SimpleID without sending your password in plain text.  Your
  91  password is used to create a cryptographic digest, which is then sent to
  92  the SimpleID server and verified.
  93  
  94  As a result, you need to be aware of two things:
  95  
  96  1.  You browser must have JavaScript switched on in order to use the new
  97      log in system.  If JavaScript is not switched on, SimpleID reverts to the
  98      "legacy" log in system used in previous versions, subject to the important
  99      point below.
 100      
 101  2.  By default, SimpleID version 0.6 will not accept logins under the legacy
 102      system.  You can override this by putting the following line in your
 103      config.inc:
 104      
 105      define('SIMPLEID_ALLOW_LEGACY_LOGIN', true);
 106      
 107      It is STRONGLY RECOMMENDED that you DO NOT switch the legacy login system
 108      on, as it is substantially less secure than the new login system.  Use this
 109      ONLY if your browser does not support JavaScript.
